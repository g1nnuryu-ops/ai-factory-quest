## 🖨 인쇄 방법 (양면/듀플렉스)

1. `명함_RYU_GEONWOO_2026-07-16_v1.html` 파일을 **크롬(Chrome)** 에서 엽니다.
2. 화면의 **「🖨 인쇄 / PDF로 저장」** 버튼을 누르거나 `Ctrl/Cmd + P` 를 누릅니다.
3. 인쇄 대화상자에서:
   - **용지 크기:** 90 × 54 mm (사용자 지정) — PDF 저장 시 자동 적용됩니다.
   - **여백(Margins):** 없음(None)
   - **배경 그래픽(Background graphics):** 켜기 — QR·색상이 정확히 인쇄됩니다.
   - **배율(Scale):** 100% (기본값 / "실제 크기")
4. 출력 결과는 **정확히 2페이지**:
   - **1페이지 = 앞면(Identity)**
   - **2페이지 = 뒷면(QR)**
5. **양면 인쇄:** 프린터의 양면(듀플렉스) 옵션에서 **짧은 가장자리(Short edge) 넘김** 을 권장합니다(가로 명함 기준 앞뒤 방향 일치).
6. 실제 명함 인쇄소 발주 시 **상하좌우 3mm 재단여백(bleed)** 을 추가로 요청하세요. 현재 파일은 재단선 없는 90×54mm 정합 기준이며, 모든 텍스트는 안전여백 ≥4mm 안쪽(좌 7mm / 상 6.4mm / 하 6mm 등)에 배치되어 있습니다.

> ✔ 검증: 헤드리스 크롬 인쇄 결과 PDF가 **2페이지 · MediaBox 255.12 × 153.12 pt (= 90 × 54 mm)** 로 확인되었습니다.

## 🛠 기술 메모

- CDN 기반 **React 18 UMD + Babel 7.26.4(고정)**. Babel 8은 인라인 JSX(`type="text/babel"`)를 깨뜨려 화면이 백지가 되므로 **반드시 7.x 고정**.
- QR base64 data URI는 길기 때문에 `text/babel` 블록 **밖** 의 일반 `<script>` 에서 `window.QR_URI` 전역으로 선언하고 React에서 읽습니다(대용량 문자열이 Babel 파싱을 깨는 문제 회피).
- 인쇄 CSS: `@page { size: 90mm 54mm; margin: 0 }`, `.card { width: 90mm; height: 54mm }`, `print-color-adjust: exact`. 화면 미리보기는 `transform: scale(2.6)` 로 확대(인쇄 시 해제).
- 한글 줄바꿈은 전 구간 `word-break: keep-all`.
- 미리보기 캡처용으로 `?face=front` / `?face=back` 쿼리를 지원합니다(단일 면만 렌더 — 스크린샷 전용, 기본 화면은 앞·뒷면 모두 표시).
