# CARD — 인스타그램 광고 카드 제작 규격

> **작성일** 2026-07-19 · 캔버스 1080×1080 (1:1) · 컨셉: **브랜드 톤앤매너 승계 + 한 줄 카피 + 여백**
> 렌더 파이프라인(HTML → headless Chrome → PNG)은 옆 폴더 [`../youtube_thumnail/thumnail.md`](../youtube_thumnail/thumnail.md) 에서 이미 검증된 방식을 그대로 승계한다.
> ⚠️ 다만 **디자인 방향은 정반대다.** 썸네일은 "소리 지르기"(굵은 외곽선·강조색 3개), 광고 카드는 "절제"(여백·색 3개 이하·외곽선 없음). 규격을 섞지 말 것.

## 0. 목표와 제약

| 항목 | 내용 |
|---|---|
| **무엇** | 인스타그램 피드 광고 카드 1장 (1080×1080, 1:1) |
| **입력** | 브랜드 인스타그램 **프로필 링크 1개** |
| **성공 기준** | ① 브랜드 계정 피드에 섞어놨을 때 **한 장처럼 보일 것** ② 피드 크기(≈400px)에서 카피가 읽힐 것 |
| **구성** | **한 줄 카피 + 비주얼 + CTA** — 이 3개가 전부. 4번째 요소를 넣고 싶으면 뭔가를 빼라 |
| **카피** | **3~7어절**, 한 줄 |
| **색** | 메인 1~2 + 보조 1 = **최대 3색** (§7) |
| **여백** | 잉크 커버리지 **30% 이하** (§5) — 고급스러움은 여기서 나온다 |
| 🔴 **반드시** | 카피는 **CSS 텍스트**로 얹는다. AI 이미지 모델로 한글을 생성하면 **반드시 깨진다** (이 저장소 반복 경험칙) |
| 🔴 **반드시** | 색·폰트·슬로건은 §2에서 뽑은 `brand.json` 값만 쓴다. **눈대중으로 "비슷한 색" 쓰지 말 것** |
| 🔴 **금지** | 글자 외곽선(`-webkit-text-stroke`). 사진 위 대비는 **스크림**으로 만든다 (§8) |
| 🔴 **금지** | 실존 인물 얼굴을 AI로 합성 · API 키를 문서/코드에 기록 (§11 보안) |

### 플랫폼 제한 (인스타그램 측, 변동 가능 — 집행 전 재확인)

| 항목 | 값 |
|---|---|
| 피드 업로드 최대 폭 | **1080px** ← 더 크게 만들어봐야 다운스케일된다. **1080이 정답** |
| 지원 비율 | 1:1 · 4:5 · 1.91:1 (본 규격은 **1:1**) |
| 재압축 | 업로드 시 **강하게 재압축된다.** 미세한 그라데이션·필름 그레인은 **띠(banding)로 뭉개진다** → 평면 색·단순 그라데이션 권장 |
| 색공간 | **sRGB.** AdobeRGB로 내보내면 채도가 죽는다 |
| 텍스트 비중 | 과거 "이미지 내 텍스트 20% 룰"은 폐지됐지만, 텍스트 과다는 **여전히 도달에 불리**하다고 안내된다. 어차피 §5 여백 규칙을 지키면 자동 충족 |

## 2. 1단계 — 브랜드 인스타그램 톤앤매너 파악

> **이 단계를 건너뛰면 광고 카드가 아니라 그냥 예쁜 카드가 된다.**
> 산출물은 `brand/BRAND.md`(사람이 읽는 리포트) + `brand/brand.json`(카드가 읽는 값) **2개**다.

### 2-1. 수집

로그인된 인스타 세션에서 Playwright로 진행한다.

```
1) browser_navigate → https://www.instagram.com/<브랜드계정>/
2) browser_take_screenshot → brand/grid/00-profile.png        ← 프로필 상단(bio·슬로건)
3) 스크롤하며 그리드 스크린샷 2~3장                              ← 피드 '전체 인상'이 여기 다 있다
4) 대표 게시물 9~12개 열어서 이미지 저장 → brand/grid/NN-*.jpg
```

수치는 내부 API로 같이 받는다 (`browser_evaluate` 안에서 `fetch`, `credentials:'include'`):

```js
// 모든 요청에 이 헤더 필요
const H = { 'x-ig-app-id': '936619743392459' };

// 프로필: 팔로워 / 카테고리 / bio / user_id
await (await fetch(`/api/v1/users/web_profile_info/?username=${u}`, {headers:H, credentials:'include'})).json();
//   → data.user.edge_followed_by.count / .category_name / .biography / .id
//   ⚠️ edge_owner_to_timeline_media.edges 는 이제 빈 배열이다. 게시물은 아래 feed로 따로 받는다

// 최근 게시물: 반응이 좋은 컷 = 브랜드가 밀고 있는 톤
await (await fetch(`/api/v1/feed/user/${userId}/?count=12`, {headers:H, credentials:'include'})).json();
//   → items[].like_count / .comment_count / .taken_at / .caption.text
//   taken_at 내림차순 정렬 후 상위 9개만 사용 (고정 게시물 왜곡 방지)
```

⚠️ **Rate limit** — 요청 사이 1.5~1.8초 간격. 계정별 try/catch로 부분 실패 격리.

### 2-2. 관찰 축 — 이 7가지를 기록한다

| 축 | 어떻게 보는가 | 기록 예시 |
|---|---|---|
| **컬러** | 대표 컷 9~12장 색 추출(아래 스크립트) + 그리드 스크린샷 전체 인상 | 메인 `#1B4332`, 서브 `#D8C3A5`, 배경 `#F7F5F2` |
| **명도 인상** | 그리드가 전체적으로 밝은가 어두운가 | "하이키 · 배경 거의 흰색" |
| **채도** | 쨍한가 뮤트인가 | "저채도, 원색 없음" |
| **폰트 인상** | 브랜드가 이미지에 얹은 글자를 본다 — 세리프/산세리프, 굵기, 자간 | "얇은 산세리프 + 넓은 자간, 전부 소문자" |
| **슬로건·상투구** | bio 첫 줄, 반복 해시태그, 캡션 반복 어구 | `#오늘도_한잔`, "매일 아침 6시" |
| **말투** | 존댓말/반말, 이모지 밀도, 문장 길이 | "짧은 평서문 + 이모지 0~1개" |
| **여백 습관** | 그리드가 꽉 찬 편인가, 비어 있는 편인가 | "피사체 작게, 여백 많음 → 카드도 여백형(§4-C)" |

**색 추출 스크립트** (대표 컷을 `brand/grid/`에 모아둔 뒤 실행):

```bash
py -c "
from PIL import Image
import glob, collections
cnt = collections.Counter()
for f in glob.glob('brand/grid/*.jpg') + glob.glob('brand/grid/*.png'):
    im = Image.open(f).convert('RGB').resize((80,80))
    q  = im.quantize(colors=6, method=Image.Quantize.MEDIANCUT).convert('RGB')
    cnt.update(q.getdata())
for (r,g,b), n in cnt.most_common(20):
    mx, mn = max(r,g,b), min(r,g,b)
    tag = 'GREY' if mx-mn < 18 else ''      # 무채색은 배경일 가능성
    print(f'#{r:02X}{g:02X}{b:02X}  {n:6d}  {tag}')
"
```

> **읽는 법:** 상위 몇 개는 보통 배경(흰색·베이지·검정)이다. 그건 **배경색 후보**로 두고,
> 무채색이 아닌 것 중 상위 1~2개가 **메인 컬러**, 그다음 하나가 **보조 컬러**다.
> 숫자만 믿지 말고 **그리드 스크린샷을 눈으로 보고** 교차 검증할 것.

### 2-3. 산출물 — `brand/brand.json`

카드 HTML은 **이 파일의 값만** 쓴다. 하드코딩 금지.

```json
{
  "handle": "@브랜드계정",
  "scanned_at": "2026-07-19",
  "slogan": "bio에서 가져온 한 줄",
  "voice": "존댓말 / 짧은 평서문 / 이모지 없음",
  "color": {
    "main":   "#1B4332",
    "main2":  null,
    "accent": "#D8C3A5",
    "bg":     "#F7F5F2",
    "ink":    "#22201D"
  },
  "font": {
    "display":   "Gowun Batang",
    "text":      "Pretendard",
    "substitute_reason": "브랜드 실제 폰트는 유료 서체로 추정 — 인상이 가장 가까운 웹폰트로 대체"
  },
  "layout_hint": "C-여백형",
  "photo_tone": "하이키 · 저채도 · 자연광 · 배경 단순"
}
```

⚠️ **브랜드 실제 폰트가 유료 서체면** 시각적으로 가장 가까운 웹폰트로 대체하고 `substitute_reason`에 **반드시 남긴다.** 나중에 "왜 다르지?"의 답이 된다.

## 3. 캔버스와 세이프존

```
1080 × 1080 px  (1:1)
바깥 여백: 상하좌우 각 96px  (프리미엄 지향이면 120~140px)
작업 해상도: HTML 1080×1080 + deviceScaleFactor 1
```

```
┌────────────────────────────────────────────┐  y=0
│ ← 96px                                     │
│    ┌──────────────────────────────────┐    │
│    │                                  │    │
│    │           안전 영역                │    │
│    │     (로고·카피·CTA 전부 이 안에)     │    │
│    │                                  │    │
│    │                                  │    │
│    └──────────────────────────────────┘    │
│                                            │
└────────────────────────────────────────────┘  y=1080
        ⚠️ 캐러셀이면 하단 중앙에 인디케이터 점이 근접
```

**크롭 현실**
- 피드에서 1:1은 **잘리지 않고 전부 보인다** — 1:1을 쓰는 가장 큰 이유다
- 프로필 **그리드**에서도 1:1은 중앙 정사각 크롭이라 손실 없음
- ⚠️ **스토리(9:16)로 재활용하면** 상하단이 UI에 먹힌다 → 핵심 요소는 **세로 중앙 60% 안**에 모아두면 재활용까지 커버된다

### 축소 생존 계산

| 디자인상 크기 | 피드 ≈400px (÷2.7) | 그리드 ≈120px (÷9) | 판정 |
|---|---|---|---|
| 140px | 52px | 16px | 임팩트형 상한 |
| **110px** | **41px** | **12px** | ✅ 임팩트형 메인 카피 |
| **84px** | **31px** | **9px** | ✅ **미니멀형 메인 카피 하한** |
| 56px | 21px | 6px | 서브 카피 한계 |
| 34px | 13px | 4px | CTA·계정명 최소 |
| 24px | 9px | 3px | ❌ 안 읽힘. 쓰지 말 것 |

> **핵심:** 고급스러움은 글자를 **키워서** 나오지 않는다. 84px + 넓은 여백이 140px + 빽빽함보다 비싸 보인다.
> 다만 **84px 아래로는 내려가지 말 것** — 피드에서 안 읽히면 디자인이 아니라 장식이다.

## 4. 레이아웃 — 한 줄 카피 + 비주얼 + CTA

세 구조 중 §2에서 파악한 브랜드 성격에 맞는 것을 고른다.

### ✅ A — 하단 카피 밴드 (기본안 / 제품·메뉴 광고)

비주얼이 주인공이고 카피가 받쳐주는 구조. 실패 확률이 가장 낮다.

```
┌──────────────────────────────────────┐
│  @brand                              │  ← 로고/핸들, 96px 여백 안
│                                      │
│         🖼  비주얼 (풀블리드)           │  ← y 0→680, 63%
│                                      │
│   ▽ 하단 120px 배경색으로 페이드         │
├──────────────────────────────────────┤  y=680
│                                      │
│   오늘의 온도, 딱 한 잔.                │  ← 메인 카피 (1줄)
│                                      │
│   ── 프로필 링크에서 →                  │  ← CTA
└──────────────────────────────────────┘  y=1080
```

- 비주얼 60~65%, 카피 밴드 35~40%
- 경계는 **배경색으로 사라지는 그라데이션** 120px — 딱 잘린 선은 싸 보인다
- 카피는 **왼쪽 정렬** (가운데 정렬은 C안에서만)

### B — 풀블리드 + 스크림 (분위기·브랜딩 광고)

사진이 압도적으로 좋을 때. 사진 위에 글자를 얹되 **반드시 스크림**을 깐다.

```
┌──────────────────────────────────────┐
│  @brand                              │
│                                      │
│        🖼  비주얼 전면                  │
│                                      │
│   여름은, 아직 안 끝났다.               │  ← 사진 위 · 스크림 필수
│   ── 자세히 보기                       │
└──────────────────────────────────────┘
```

- 스크림: `linear-gradient(180deg, transparent 40%, rgba(0,0,0,.55) 100%)` — 글자 뒤에만, 사진 전체를 어둡게 덮지 않는다
- 🔴 글자 외곽선 대신 스크림. 외곽선은 즉시 "썸네일"로 보인다
- 사진의 **비어 있는 쪽**에 글자를 놓는다. 피사체 위에 겹치면 둘 다 죽는다

### C — 여백형 (프리미엄·미니멀 / §2에서 "여백 많음"이 나왔을 때)

가장 고급스럽지만 **비주얼이 좋아야** 성립한다. 어설프면 그냥 빈 카드가 된다.

```
┌──────────────────────────────────────┐
│                                      │
│              @brand                  │
│                                      │
│         ╭──────────╮                 │  ← 비주얼을 작게, 중앙
│         │   🖼      │                 │     (제품 컷아웃 / 정사각 사진)
│         ╰──────────╯                 │
│                                      │
│        가 볍 게 ,  깊 게 .            │  ← 자간 넓게
│                                      │
│           자세히 보기 →                │
└──────────────────────────────────────┘
```

- 비주얼이 화면의 **35~45%만** 차지 — 나머지는 전부 여백
- 가운데 정렬 + `letter-spacing: 0.08em~0.15em`
- 색은 **배경 + 잉크 + 보조 1** 이면 충분. 메인 컬러도 아껴 쓴다

## 5. 여백 규칙 — 이 문서의 핵심

> "고급스럽게"는 감성 형용사가 아니라 **측정 가능한 조건**이다. 아래를 숫자로 지키면 자동으로 그렇게 된다.

| 규칙 | 값 | 왜 |
|---|---|---|
| **바깥 여백** | 최소 **96px** (전체의 8.9%) · 프리미엄 **120~140px** | 가장자리에 붙은 요소는 즉시 전단지처럼 보인다 |
| **잉크 커버리지** | 텍스트+그래픽+CTA가 덮는 면적 **30% 이하** | 사진 영역은 제외. 이게 넘으면 뭔가 빼야 한다 |
| **요소 개수** | **5개 이하** (로고 / 비주얼 / 카피 / CTA / 핸들) | 6번째가 생각나면 5개 중 하나를 지운다 |
| **카피 줄 수** | **1줄** (길면 2줄까지, 3줄은 실패) | 3줄이 필요하면 카피가 긴 것이다 (§6) |
| **요소 간 간격** | 인접 요소 사이 ≥ **작은 쪽 글자 크기 × 1.5** | 붙어 있으면 하나로 읽힌다 |
| **정렬축** | **1~2개만** | 왼쪽·가운데·오른쪽을 섞으면 산만해진다 |
| **카피 ↔ CTA 간격** | ≥ 카피 글자 크기 × 2 | 붙으면 CTA가 카피의 일부처럼 보인다 |

**커버리지 셀프체크** — 눈으로 안 잡히면 재본다:

```bash
py -c "
from PIL import Image
im = Image.open('renders/card.png').convert('L').resize((216,216))
px = list(im.getdata())
# 배경 밝기 기준(밝은 배경 가정). 어두운 배경이면 부등호를 뒤집을 것
ink = sum(1 for v in px if v < 200)
print(f'대략적인 잉크 커버리지: {ink/len(px)*100:.1f}%   (목표 30% 이하)')
"
```

## 6. 카피 — 3~7어절, 한 줄

| 요소 | 규격 |
|---|---|
| **길이** | **3~7어절** (띄어쓰기 기준) · 한글 **6~16자** |
| **줄 수** | 1줄 (부득이하면 2줄) |
| **문장부호** | 마침표 1개까지. 느낌표는 **임팩트형에서만** |
| **금지** | 설명하려 들지 말 것. 스펙·가격·날짜는 **카피가 아니라 보조 정보** |

### 좋은 카피의 조건

- **동사 또는 감각어가 있다** — "프리미엄 원두 사용"(❌ 명사 나열) vs "오늘의 온도, 딱 한 잔."(✅)
- **브랜드 말투와 같다** — §2에서 뽑은 `voice`가 반말이면 카피도 반말
- **읽고 나서 뭘 하는지 안다** — 그게 안 되면 CTA가 대신 못 해준다

### 패턴 5개 (막힐 때)

| 패턴 | 예시 |
|---|---|
| 감각 + 선언 | `여름은, 아직 안 끝났다.` |
| 숫자 1개 | `단 3일, 딱 한 번.` |
| 대비 | `가볍게, 깊게.` |
| 질문 | `오늘 뭐 마셔요?` |
| 시간·상황 지정 | `퇴근길에 한 잔.` |

🔴 **어절 수가 넘치면 폰트를 줄이지 말고 카피를 줄인다.** 광고 카드가 실패하는 1순위 원인은 글자가 많은 것이다.

### 보조 정보 (선택)

가격·기간·조건은 **넣더라도 1줄, 34~44px**, 메인 카피의 절반 이하 크기로. 넣을 자리가 애매하면 **캡션으로 빼는 게 정답이다.**

## 7. 색 — 메인 1~2 + 보조 1

`brand.json`의 값만 쓴다.

| 슬롯 | 개수 | 역할 | 면적 |
|---|---|---|---|
| **배경** | 1 | 카드 바탕 (브랜드 뉴트럴·흰색·딥톤) | **~60%** |
| **메인** | **1~2** | 브랜드 정체성. 큰 면·로고·핵심 단어 | **~30%** |
| **보조** | **1** | **CTA 또는 강조 단어 하나** | **~10%** |
| 잉크 | 1 | 본문 텍스트 (거의 무채색) | 색 수에 포함 안 함 |

### 규칙

- 🔴 **보조색은 한 군데만.** CTA에 썼으면 강조 단어에는 쓰지 않는다. 두 군데 이상 쓰는 순간 보조색이 아니라 세 번째 메인이 된다
- 메인을 2개 쓸 경우 **명도 차를 크게** 벌린다 (딥 + 라이트). 명도가 비슷한 두 색은 경계가 진동한다
- **채도 높은 색끼리 맞붙이지 않는다.** 붙일 거면 사이에 배경색이나 무채색을 끼운다
- 그라데이션은 **같은 계열 2스톱**까지. 인스타 재압축에서 다색 그라데이션은 띠가 생긴다

### 대비 기준 (검수 가능한 숫자)

| 대상 | 최소 명도대비 |
|---|---|
| 메인 카피(대형 텍스트) | **3:1** |
| CTA·보조 정보(작은 텍스트) | **4.5:1** |

사진 위 텍스트는 **가장 밝은 지점 기준**으로 잰다. 평균으로 재면 밝은 부분에서 글자가 사라진다.

### 크기 규격 (1080×1080 기준)

| 요소 | 임팩트형 | 미니멀형 |
|---|---|---|
| 메인 카피 | 110~140px | **84~96px** |
| 보조 정보 | 44~56px | 34~44px |
| CTA | 40~48px | 34~40px |
| 로고·핸들 | 34~44px | 30~36px |

### CSS 기본형

```css
.copy {
  font-family: 'Gowun Batang', serif;   /* brand.json 의 font.display */
  font-size: 92px;
  line-height: 1.25;                     /* 카드는 썸네일보다 넉넉하게 */
  letter-spacing: -0.01em;               /* 미니멀 C안은 +0.08em~0.15em */
  color: var(--ink);
  word-break: keep-all;                  /* 🔴 한국어 단어 중간 줄바꿈 방지 */
  text-wrap: balance;                    /* 2줄이 될 때 줄길이를 고르게 */
}
```

- 🔴 **`word-break: keep-all`** 빠뜨리면 "한 잔"이 "한/잔"으로 쪼개진다
- 🔴 **외곽선 금지.** 사진 위라면 스크림(§4-B). 정 필요하면 `paint-order: stroke fill`을 **반드시** 같이 쓴다 — 없으면 한글 자소가 뭉개진다 (썸네일 문서에서 실측 확인된 사항)
- 그림자는 쓰더라도 아주 약하게: `text-shadow: 0 2px 20px rgba(0,0,0,.25)`. 진한 그림자는 즉시 저렴해 보인다

## 9. CTA

인스타 피드 이미지 안의 CTA는 **클릭되지 않는다.** 그러므로 CTA는 버튼 흉내가 아니라 **"어디로 가야 하는지"를 알려주는 문장**이다.

| 항목 | 규격 |
|---|---|
| **개수** | 🔴 **1개.** 두 개면 둘 다 안 눌린다 |
| **문구** | `프로필 링크에서` · `DM으로 문의` · `저장해두세요` · `@계정명` — **경로를 지정**할 것 |
| **형태** | 밑줄 텍스트 + `→` (권장) · 얇은 테두리 pill · 채운 pill(보조색, 임팩트형만) |
| **크기** | 34~48px. **메인 카피보다 크면 안 된다** |
| **위치** | 하단. 메인 카피와 **글자 크기 × 2 이상** 띄운다 |
| **색** | 보조색 1개 — §7에서 보조색을 여기 썼으면 다른 곳엔 안 쓴다 |

```css
.cta {
  font-family: 'Pretendard', sans-serif;
  font-weight: 600;
  font-size: 40px;
  color: var(--accent);
  border-bottom: 2px solid var(--accent);
  padding-bottom: 10px;
}
```

❌ `지금 클릭!` — 클릭 안 된다
✅ `프로필 링크에서 →` — 실제로 갈 수 있는 곳

## 10. 비주얼 소재

### 우선순위

1. **브랜드가 실제로 올린 사진** — 톤 일치도가 압도적. §2에서 이미 받아뒀다. 단 **사용 권한 확인 필수** (§11)
2. **fal.ai 생성** — 브랜드 톤을 프롬프트로 재현
3. **제품 컷아웃** — `fal-ai/birefnet` 으로 배경 제거 후 배경색 위에 얹기 (C안에 적합)

### fal.ai 생성 — `brand.json`의 `photo_tone`을 프롬프트로 번역

```js
// ../movie_poster/.env 의 FAL_KEY 재사용 (커밋 금지)
import { readFile, writeFile } from "node:fs/promises";
const FAL_KEY = (await readFile("../movie_poster/.env", "utf8")).match(/FAL_KEY=(.+)/)[1].trim();

const res = await fetch("https://fal.run/fal-ai/flux/dev", {
  method: "POST",
  headers: { Authorization: `Key ${FAL_KEY}`, "Content-Type": "application/json" },
  body: JSON.stringify({
    prompt: PROMPT,
    image_size: { width: 1080, height: 1080 },
    num_images: 3,
  }),
});
const out = await res.json();
const bin = Buffer.from(await (await fetch(out.images[0].url)).arrayBuffer());
await writeFile("assets/visual.jpg", bin);
```

**프롬프트 작성 요령** (이 저장소에서 실패로 배운 것)

- ❌ `shallow depth of field` / `85mm` / `bokeh` + `seamless studio backdrop` → **피사체까지 뭉개진다.** 결과물이 수상하게 작은 파일(~110KB)로 나오면 이 함정에 빠진 것
- ✅ **환경 + 방향광**으로 쓴다: `on a marble table, afternoon light from a window, 45° angle`, `overhead flat lay, daylight raking from the left`
- ✅ 항상 붙일 것: `absolutely no text, no lettering, no logos, no watermark` — 한글은 **무조건 깨진다**
- ✅ 생성 후 **파일 크기를 먼저 본다.** 선명한 컷은 보통 250KB+ 다. 그리고 **반드시 Read로 열어서 눈으로 확인**

### 배경 제거 (컷아웃)

```js
// 🔴 birefnet 을 쓴다. rembg 는 채도 높은 배경 위에서 밝은 헤일로가 남는다
await fetch("https://fal.run/fal-ai/birefnet", {
  method: "POST",
  headers: { Authorization: `Key ${FAL_KEY}`, "Content-Type": "application/json" },
  body: JSON.stringify({ image_url: SOURCE_URL }),   // 공개 URL 또는 data: URI
});
```

⚠️ 컷아웃 품질은 **알파 통계가 아니라 채도 높은 배경 위에 합성해서** 눈으로 판단한다. 반투명 픽셀이 많은 쪽이 더 부드러운 게 아니라 **헤일로**인 경우가 있다.

## 11. 파이프라인

```
card.html  (1080 × 1080, 인라인 CSS, 이미지는 base64 data URI)
     │
     ├─ 웹폰트: brand.json 의 font (CDN)
     ├─ 색: brand.json 의 color 를 CSS 변수로
     ├─ 카피는 실제 CSS 텍스트
     │
     ▼
headless Chrome --screenshot --window-size=1080,1080
     │                       --force-device-scale-factor=1
     │                       --virtual-time-budget=10000
     ▼
renders/card_<brand>_2026-07-19_v1.png   (1080 × 1080)
     │
     ▼  업로드용
card.jpg  (quality 92, sRGB)
```

```powershell
# 🔴 PowerShell에서 실행할 것 (Bash로 하면 경로가 깨진다)
& "C:\Program Files\Google\Chrome\Application\chrome.exe" `
  --headless=new --disable-gpu --hide-scrollbars `
  --screenshot="$PWD\renders\card.png" `
  --window-size=1080,1080 --force-device-scale-factor=1 `
  --virtual-time-budget=10000 `
  --user-data-dir="$env:TEMP\cp-card-01" `
  "$PWD\card.html"
```

**주의 (전부 이 저장소에서 직접 겪은 것)**

- 🔴 **`--user-data-dir`은 호출마다 새 이름**으로. 재사용하면 **아무 오류 메시지 없이 PNG가 안 만들어진다**
- 🔴 **`--virtual-time-budget=10000` 없으면** 웹폰트 로드 전에 캡처돼 **맑은 고딕으로 폴백**된다. 조용히 실패하므로 결과물을 눈으로 봐야 안다
- PowerShell에서 chrome 실행 시 `NativeCommandError`가 떠도 **파일은 정상 생성**된다. stderr를 에러로 오해하지 말 것
- 이미지는 **base64 data URI**로 넣어야 `file://`에서 안전하다. 큰 base64는 스크립트 밖으로 뺄 것
- 출력이 1080이 아니면 `--force-device-scale-factor=2` + `--window-size=540,540` 으로 바꾸고 **CSS 값을 전부 절반**으로 (스펙 px ÷ 2)
- PowerShell `Remove-Item` + `C:\Program` 경로를 **같은 명령에 섞으면** 샌드박스가 조용히 막는다. 정리는 따로 실행

**검수·내보내기**

```bash
# 규격 확인
py -c "from PIL import Image; im=Image.open('renders/card.png'); print(im.size, im.mode)"

py -c "from PIL import Image; im=Image.open('renders/card.png').convert('RGB'); im.save('renders/card.jpg', quality=92, optimize=True)"
```
