# RAPA 증명서 발급 시스템

한국전파진흥협회(RAPA) AX·DX 교육센터용 증명서 발급 웹앱.
**서버 없이** 동작합니다 — 정적 `index.html` + Google Apps Script 웹앱 + 공유 구글시트.

```
[index.html · 애플풍 UI]  ──fetch──▶  [Apps Script 웹앱(Code.gs)]  ──▶  [공유 시트 gid 210882555]
   좌: 입력폼 / 우: 증명서 실시간 미리보기        ├─ RAPA 과정목록 + HRD-Net 상세/수강생 명단(이름·생년월일)
                                              └─ 발급 1건 = 시트 1행 누적
```

## 파일
| 파일 | 역할 |
|---|---|
| `index.html` | 단일 파일 React(CDN) 앱. 증명서 5종(수강확인증/수료증/참여확인서/프로젝트 우수상/모범상), 실시간 미리보기, 인쇄·PDF 저장. RAPA 로고·직인 내장 |
| `Code.gs` | Google Apps Script 웹앱(백엔드). 과정목록 + 수강생 명단(_4) + 과정상세(_2) + 시트 적재 |
| `Sheet1.xlsx` | RAPA 운영 과정목록 원본(과정ID/회차/기간). `Code.gs`의 `RAPA_COURSES` 생성에 사용 |
| `한국전파진흥협회 로고.png` | 증명서 상단 로고 (index.html 에 base64 로 내장) |
| `협회 직인.jpg` | **커밋 금지.** 협회 직인 원본. `config.js` 의 `SEAL_SRC` 로 주입한다 (`.gitignore` 처리됨) |
| `config.example.js` | 프런트 설정 템플릿. 복사해서 `config.js` 를 만든다 |
| `config.js` | **커밋 금지.** 웹앱 URL 실제 값 (`.gitignore` 처리됨) |
| `.env` | **커밋 금지.** HRD-Net `authKey` (`.gitignore` 처리됨) |

> ⚠️ **이 폴더는 `quest/` 안에 있어 공개 저장소 `ai-factory-quest` 로 자동 동기화된다.**
> 키·시트ID·웹앱 URL·훈련생 실명이 든 파일을 코드나 폴더에 그대로 두지 말 것.
> 자세한 내용은 아래 [보안](#보안) 참조.

## 발급되는 데이터 (시트 컬럼)
`수료순서 · 증명서 번호(RAPA26-AXDX-####) · 교육생 이름 · 생년월일 · 증명서 종류 · 교육과정명 · 담당부서 · 담당자`
— 번호와 순서는 백엔드가 마지막 행을 보고 자동 증가시킵니다.

---

## 1) 백엔드 배포 (최초 1회, 시트 편집 권한 계정으로)
1. 대상 구글시트 열기 → 상단 **확장 프로그램 → Apps Script**
2. 기본 코드 지우고 **`Code.gs` 전체 내용**을 붙여넣기 → 저장(Ctrl+S)
3. **⚙️ 프로젝트 설정 → 스크립트 속성**에 아래 값 등록 — **코드에 직접 넣지 말 것**

   | 속성 이름 | 값 | 필수 |
   |---|---|---|
   | `SHEET_ID` | 발급대장 구글시트 ID (URL의 `/d/` 와 `/edit` 사이) | ✅ |
   | `HRD_AUTH_KEY` | HRD-Net 기관키 — `.env` 의 `api_KEY` 값 | ✅ |
   | `CUSTOM_API_KEY` | RAPA 내부 API 키 (안 쓰면 생략) | — |

4. 편집기에서 **`_check_config()` 실행** → 로그에 시트명·다음 번호가 찍히면 정상
5. 우상단 **배포 → 새 배포** → 유형 톱니바퀴 → **웹 앱**
6. 설정 → **실행: 나(본인)**, **액세스 권한: 모든 사용자** → **배포**
7. 권한 승인(처음 1회). "확인되지 않은 앱" 경고 시 **고급 → (안전하지 않음) 이동**
8. 생성된 **웹 앱 URL** 복사 — `https://script.google.com/macros/s/.../exec`

> 배포 후 `Code.gs`를 수정하면 **배포 관리 → 편집(연필) → 버전: 새 버전 → 배포**로 갱신해야 반영됩니다.
> 스크립트 속성은 프로젝트에 저장되므로 재배포해도 다시 넣을 필요가 없습니다.

## 2) 프런트 연결
`config.example.js` 를 같은 폴더에 **`config.js`** 로 복사하고 URL과 직인을 채웁니다:
```js
window.RAPA_CONFIG = {
  WEBAPP_URL: "https://script.google.com/macros/s/.../exec",
  SEAL_SRC:   "data:image/jpeg;base64,..."   // 협회 직인 (아래 참고)
};
```
직인 data URL 은 로컬 원본에서 만듭니다 (PowerShell):
```powershell
$b = [Convert]::ToBase64String([IO.File]::ReadAllBytes("협회 직인.jpg"))
"data:image/jpeg;base64,$b" | Set-Clipboard
```
- `config.js` 가 없으면 **데모 모드**(샘플 데이터)로 동작합니다. 콘솔의 `config.js` 404는 정상입니다.
- `SEAL_SRC` 가 비면 직인 자리에 **CSS placeholder**(붉은 사각 인장)가 렌더링됩니다. 실물 발급에는 반드시 채워야 합니다.
- 채우면 실제 모드: 과정·수강생 명단 자동조회 + 발급 시 시트에 행 누적.
- `config.js` 는 `.gitignore` 처리되어 **커밋되지 않습니다.** `index.html` 에 URL을 직접 적지 마세요.

## 3) 실행 / 배포
- 로컬 확인: VS Code **Live Server**, 또는 폴더에서 `npx serve .` 후 표시 URL 열기.
- 사내 배포: `index.html` 한 파일을 정적 호스팅(사내 웹서버, Netlify, Vercel, GitHub Pages 등)에 올리면 됩니다.

## 사용 흐름
1. **증명서 종류** 선택 (수강확인증 등 5종)
2. **교육과정** 선택 (RAPA 운영 과정 목록) → 선택하면 **교육기간·일수/시간·사업명·담당자 자동 채움** (모두 수정 가능)
3. **교육생** 선택 → 해당 과정 **수강생 명단(이름·생년월일)이 `_4`로 자동 조회** → 선택만 하면 채워짐 (명단 없을 땐 직접 입력/붙여넣기)
4. **증명서 번호**는 자동(편집 가능), **발급일자** 선택
5. **발급 & 시트 기록** → 시트에 1행 누적 + 번호 자동 증가
6. **인쇄/PDF 저장** → 공식 양식 그대로 A4 출력

## API 메모 (검증 완료, 2026-06)
- `.env`의 키 `yEV52...`는 **HRD-Net 구 OpenAPI(`hrd.work24.go.kr/.../HRDPOA60_1.jsp`, `_2.jsp`)**에서 정상 동작 확인.
  (신형 `www.work24.go.kr/cm/openApi/...` 엔드포인트에는 미인증이라 구 JSP 엔드포인트 사용.)
- 응답은 `{"returnJSON":"{...}"}` 이중 래핑 — 백엔드에서 자동 파싱.
- **수강생 명단(이름·생년월일)**: 공개 catalog API엔 없지만, `.env` 키가 **RAPA 기관 키**라 **기관전용 `_4.jsp`(HRDPOA60_4)** 로 RAPA 자기 과정의 수강생 명단(이름·생년월일·출결)을 조회합니다 (`outType=2`, `srchTorgId=default`). 응답은 XML `trneList>trne_list` (`trneeCstmrNm`=이름, `lifyeaMd`=생년월일).
- **RAPA 과정목록**은 `Sheet1.xlsx`(과정ID/회차/기간)에서 생성해 `Code.gs`의 `RAPA_COURSES`에 내장. 과정 변경 시 xlsx 재공유 → 재생성(또는 시트 탭으로 이전).
- 과정 상세(일수/시간/사업명/담당자)는 선택 시 `_2.jsp`로 보강. 명단·상세 모두 RAPA 본인 기관 과정만 조회됨.

## 보안

이 폴더는 `quest/` 안에 있어 **공개 저장소 `ai-factory-quest` 로 자동 동기화**됩니다.
따라서 "커밋되는 파일"과 "로컬에만 두는 파일"의 경계가 곧 보안 경계입니다.

### 비밀값은 어디에 두는가

| 값 | 보관 위치 | 커밋? |
|---|---|---|
| HRD-Net 기관키 | Apps Script **스크립트 속성** `HRD_AUTH_KEY` / 로컬 `.env` | ❌ |
| 발급대장 시트 ID | Apps Script **스크립트 속성** `SHEET_ID` | ❌ |
| 웹앱 URL | `config.js` (`window.RAPA_CONFIG.WEBAPP_URL`) | ❌ |
| 협회 직인 이미지 | `config.js` (`SEAL_SRC`) / 원본 `협회 직인.jpg` 는 로컬 전용 | ❌ |
| 발급된 실물 증명서 PDF | 로컬 전용 (`*.pdf` 는 gitignore) | ❌ |
| 빈 양식 PDF `(양식)*.pdf` | 기준 문서라 예외적으로 커밋 | ✅ |

`Code.gs` · `index.html` 에는 **어떤 비밀값도 들어 있지 않습니다.** 하드코딩하지 마세요.

**협회 직인을 코드에서 뺀 이유:** 직인 이미지는 키가 아니지만 **공식 문서를 위조할 수 있는 기관 자산**입니다.
공개 저장소에 올라가면 누구나 내려받아 협회 명의 증명서를 만들 수 있으므로, 웹앱 URL과 동일하게 `config.js` 로만 주입합니다.
같은 이유로 **직인이 찍힌 발급물 캡처·발표 자료도 공개 트리에 넣지 않습니다.**

### 웹앱 URL을 비밀번호처럼 다뤄야 하는 이유

웹앱이 **액세스: 모든 사용자**로 배포되어 있어, URL만 알면 인증 없이

- `?action=roster&...` → 훈련생 **이름·생년월일** 조회 (HRD-Net `_4` 기관전용 데이터)
- `POST` → 발급대장에 **행 쓰기**

가 가능합니다. URL 유출 = 개인정보 유출입니다. 외부 공유·스크린샷·문서 첨부에 주의하세요.

### 자동 차단 장치

`.githooks/sync-quest.sh` 가 공개 저장소로 푸시하기 **전에** 트리 내용을 검사해,
API 키·웹앱 URL·private key 형태의 문자열이 발견되면 **동기화를 중단**합니다
(`your_`, `<...>`, `example` 같은 플레이스홀더는 오탐 제외).
차단되면 해당 값을 `.env` / `config.js` / 스크립트 속성으로 옮기고 다시 커밋하세요.

> ⚠️ 이 장치는 **공개 미러 푸시만** 막습니다. private 저장소(`ai-factory`)로의 push는 그대로 진행되므로,
> 애초에 비밀값을 코드에 쓰지 않는 것이 1차 방어선입니다.
