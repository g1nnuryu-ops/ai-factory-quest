**구성:** `index.html`(단일파일 CDN React) + `Code.gs`(Google Apps Script 백엔드) + 공유 구글시트(DB). `WEBAPP_URL` 이 실제 `/exec` URL 로 채워져 있음 → **데모가 아니라 배포된 상태.**

**이미 되는 것:**
- 증서 **5종**: 수강확인증 / 수료증 / 참여확인서 / 프로젝트 우수상 / **모범상**

- 발급 1건 = 구글시트 1행 누적, 증명서 번호 `RAPA26-AXDX-####` 자동 증가 (LockService + 중복 검사 있음)
- 브라우저 인쇄로 A4 PDF 저장

**Why 중요:** 사용자가 "교육 수료증/수강확인증/모범상을 외부 API 연결해 자동 발급하는 앱을 만들고 싶다"고 하면, 그건 **이 앱과 거의 동일한 요구사항**이다. 모르고 스코핑하면 중복 구현을 미션으로 못박게 된다.

**How to apply:** 증서·발급·HRD-Net 관련 아이디어가 나오면 **먼저 RAPA/ 를 읽어라.** 사용자는 개인 프로젝트 과제가 있는 줄 모르고 이 앱을 먼저 만들어버렸다 — 그래서 과제의 미션은 "새로 만든다"가 아니라 **"v1 을 보완한다(v2)"** 가 된다.

**v2 로 확정된 스코프 (2026-07-13):**
1. **선도기업 공동 발행 양식** — 실제 수료증(`RAPA/(양식) 수료증_LIG 3기.pdf`)은 **선도기업 + RAPA 공동 발행**인데 앱은 RAPA 로고·직인 하나만 렌더링한다. 선도기업은 본문 문장에 이름만 들어갈 뿐 로고 자리도 서명란도 없다. 선도기업은 과정마다 다름(카카오/네오위즈/LIG D&A/WINS…, 10개 안팎). **대표이사 직인은 앱이 안 찍고 인쇄 후 실물 날인**으로 확정. 적용 대상은 **수료증 + 상장류(우수상·모범상)**, 수강확인증·참여확인서는 RAPA 단독 유지.
2. **수료자 일괄 발급** — 현재 1명씩이라 40명이면 40번. `trneeSttusNm` 로 수료자 자동 필터 가능.
3. **구글시트 → 정식 DB** — 채번이 "이름 빈 첫 행 슬롯 채우기"라 행 삭제·정렬에 취약. 번호 UNIQUE 제약을 DB 스키마로.

**로고 보관 결정:** NAS 직접 연동 불가(Apps Script 는 구글 서버 실행 → 사내망 경로 없음, 브라우저 UNC/`file://` 차단, NAS 웹UI CORS 없음) → **앱에 등록하는 방식.** 단 v2 를 Node 로컬 실행으로 가면 NAS 파일 경로 직접 읽기 가능 → 배포 방식과 묶인 선택.

**미해결 리스크:** RAPA 협회 직인은 앱이 이미지로 찍는데 선도기업 직인은 실물 날인 → **한 문서에 이미지 직인 + 실물 직인 혼재가 실무·감사상 허용되는지 미확인.** 안 되면 RAPA 직인도 빼야 함.
