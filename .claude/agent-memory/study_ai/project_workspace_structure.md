This is a weekly study/course workspace (AI 공장) where each `week_N` folder holds in-class materials plus a `quest/` subfolder of homework apps. There is no separate lesson-notes markdown — the lessons ARE the example files themselves (HTML apps, recipe .md files).

- **week_1**: AI-agent / content-generation week. Markdown recipes (e.g. ham-tteokbokki.md, gambas-al-ajillo.md), a K-drama diary, plus AI-tool screenshots and zipped deliverables. No HTML/coding here — it is about using AI agents to produce text/markdown + thumbnails. quest = diary_류건우 / recipes_류건우.
- **week_2**: Web fundamentals week (HTML / CSS via Tailwind CDN / JS, then React-in-HTML via Babel). In-class folders are date-prefixed `260602`, `260602-2 … -5`, plus standalone mini-apps (260602-dday계산기, 260602-나이계산기, 260602-색상팔레트). quest = small apps (QR 코드 생성기, PDF 생성기, 더치페이 계산기, 세금계산기, 짤 생성기, 260609 고블린 with weather-app + nasa-apod-magazine).
- **week_3**: exists at workspace root and also `quest/week_3`; not yet read in detail.

**Why:** Understanding that "what was taught" must be inferred from the example/app files, not from any lecture-notes doc, since none exist.
**How to apply:** When asked "what did we learn in week N", enumerate that week's folder and read the in-class (date-prefixed) files first; treat `quest/` as homework applying the lesson. Recipe .md files in week_2 are DATA reused from week_1, not HTML lessons.
