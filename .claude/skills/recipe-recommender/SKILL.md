# Recipe Recommender — 보유 식재료 기반 레시피 추천·생성

`quest/week_4` 아래의 식재료 JSON 데이터를 읽어, **지금 집에 있는 재료**로 만들 수 있는 레시피를
추천하거나 새로 만들어 주는 스킬입니다. 대상 가정은 **3~7세 아이 2명을 포함한 4인 가족**이므로,
알레르기 안전과 아이 입맛을 항상 우선으로 고려합니다.

---

## 1단계 — 식재료 데이터 로딩 (반드시 먼저)

데이터는 `quest/week_4/*/ingredients/*.json` 에 재료당 1개 파일로 들어 있습니다.

> ⚠️ **`Glob` 도구는 쓰지 마세요.** 폴더명이 한글(`1_Skill_냉장고재료_기반_레시피_제작스킬`)이라
> Glob 도구는 유니코드 정규화 문제로 파일을 찾지 못합니다. **`Bash` 또는 `Grep` 도구를 사용하세요.**

**기본 방법 (Bash, 한 번에 전부 읽기 — 저장소 루트에서 실행):**

```bash
for f in quest/week_4/*/ingredients/*.json; do echo "=== $f ==="; cat "$f"; echo; done
```

- 파일이 안 보이면 작업 디렉터리가 저장소 루트인지 확인하세요(상대경로 기준).
- 그래도 못 찾으면 **`Grep` 도구**로 위치를 찾으세요:
  `Grep(pattern: "\"id\":", path: "quest/week_4", glob: "*.json", output_mode: "files_with_matches")`
- 특정 재료만 필요하면 `Read` 도구로 해당 파일을 직접 읽어도 됩니다.

### 데이터 스키마 (각 JSON 필드)

| 필드 | 의미 | 레시피에 쓰는 법 |
|------|------|------------------|
| `이름` / `영문명` | 재료명 | 표시·매칭 |
| `수량` / `단위` | 보유량 | 분량 가능 여부 판단 |
| `카테고리` | 채소·육류·유제품·과일·양념 등 | 영양 균형(탄수+단백+채소) 구성 |
| `보관방법` | 냉장 / 냉동 / 실온 | "냉장고 재료" vs "상온 재료" 필터 |
| `소비기한_일` | 권장 소비기한(일) | **짧은 재료 우선 소진** 추천 |
| `알레르기` | 알레르기 유발 요소(배열) | **회피·표시 필수** |
| `아이선호도` | 1~5 (조미료는 null) | 높을수록 우선, 낮으면 "숨기는" 조리법 |
| `주요영양소` | 대표 영양소(배열) | 영양 포인트 설명 |
| `대표요리` | 자주 만드는 요리(배열) | 추천 후보의 출발점 |
| `비고` | 보관·조리·유아식 팁 | 조리 팁·간 조절에 반영 |

## 2단계 — 사용자 요청·제약 파악

요청에서 다음을 파악하세요. 불명확하고 결과가 크게 갈리면 `AskUserQuestion`으로 1~2개만 짧게 확인하고,
그렇지 않으면 합리적 기본값으로 진행합니다.

- **모드**: 메뉴를 *추천*해 달라는지(여러 후보) vs 특정 요리를 *만들어*(상세 레시피) 달라는지
- **끼니/상황**: 아침·점심·저녁·간식·반찬, 손님 유무
- **대상**: 온 가족 / 아이 위주 / 어른 위주
- **제약**: 조리 시간, 인분(기본 **4인 = 어른 2 + 아이 2**), 매운맛 정도, 특정 알레르기 회피, 사용/제외할 재료
- **재료 범위**: "냉장고 재료만", "상온 재료 포함" 등 → `보관방법`으로 필터

기본값: 별다른 말이 없으면 **온 가족 저녁, 4인분, 순한 맛, 30분 내외, 보유 재료 우선**.

---

## 3단계 — 추천 또는 생성

### A. 추천 모드 ("뭐 해먹지", "메뉴 추천")
1. 보유 재료로 무리 없이 만들 수 있는 요리 **2~3개**를 골라 한 줄 요약으로 제시.
   - 각 후보: 요리명 · 핵심 재료 · 예상 시간 · 아이 적합도(한 줄).
   - 각 재료의 `대표요리`를 후보 발굴의 출발점으로 쓰되, 보유 재료 조합으로 실제 가능한지 검증.
2. `소비기한_일`이 짧은 재료를 쓰는 메뉴를 우선 추천하고 그 이유를 한 줄로 덧붙임("딸기·시금치가 곧 상해요").
3. 사용자가 하나 고르면 **B(생성 모드)**로 상세 레시피 제공.

### B. 생성 모드 (특정 요리 상세 레시피)
아래 **출력 템플릿**대로 완성된 레시피 1개를 작성합니다.

- 보유 재료 + 기본 양념(소금·간장·식용유·참기름·설탕 등 데이터에 있는 것)만으로 구성.
- 데이터에 **없는 재료가 꼭 필요하면** "추가 구매" 항목에 따로 표시(임의로 있다고 가정 금지).
- 분량은 인분에 맞춰 `수량`/`단위` 기준으로 현실적으로.

---

## 레시피 작성 원칙 (3~7세 아이 있는 4인 가족)

1. **알레르기 안전 최우선**: 각 재료 `알레르기`를 확인. 사용자가 회피 알레르겐을 말하면 그 재료를 제외하고
   대체재를 제안. 레시피에는 포함된 알레르겐을 **반드시 표시**.
2. **아이 입맛 반영**: `아이선호도`가 높은 재료를 중심으로. 선호도가 낮지만 영양가 있는 재료(예: 시금치)는
   잘게 다져 계란말이·볶음밥에 섞는 등 **거부감을 줄이는 조리법**을 함께 제안.
3. **순한 간**: 소금·설탕·매운 양념 최소화. 김치는 헹궈서 사용. 매운 어른용은 **덜어낸 뒤 따로 양념**하는
   2단계 조리를 안내.
4. **영양 균형**: 한 끼에 탄수화물 + 단백질 + 채소가 들어가도록 구성.
5. **유통기한 고려**: `소비기한_일`이 짧은 재료를 먼저 쓰도록 우선순위.
6. **현실성·안전**: 보유 재료를 벗어난 추측 금지. 아이가 먹는 음식이므로 충분히 익히는 조리(특히 육류·계란)를 명시.

## 출력 템플릿 (생성 모드)

```markdown
## 🍳 [요리명]

- ⏱️ 소요시간: 약 N분  ·  👨‍👩‍👧‍👦 N인분  ·  🌶️ 맵기: 순함/보통
- 👶 아이 적합도: ★★★★☆ — (한 줄 코멘트)

### 재료
**보유 재료** (데이터 기준)
- 계란 2개, 당근 1/2개, 밥 2공기 …

**추가 구매(있으면)**
- (없으면 "없음")

### ⚠️ 알레르기
- 난류, 우유 포함 (없으면 "해당 없음")

### 만드는 법
1. …
2. …

### 👶 아이를 위한 팁
- 매운 양념은 어른 몫만 마지막에 / 채소는 잘게 다져서 등

### 🥗 영양 포인트
- 단백질(계란)·베타카로틴(당근)으로 …
```

추천 모드에서는 위 전체 대신 후보별 **2~3줄 요약**만 제시하고, 선택 시 전체 템플릿으로 확장합니다.

## (선택) 레시피 저장

사용자가 저장을 원하면 레시피 마크다운을
`quest/week_4/1_Skill_냉장고재료_기반_레시피_제작스킬/recipes/<영문-슬러그>.md` 로 저장합니다
(`recipes/` 폴더가 없으면 생성). 요청하지 않으면 채팅에만 출력합니다.

## 동작 예시

- 사용자: "오늘 저녁 뭐 해먹지?"
  → 데이터 로딩 → 보유 재료로 가능한 메뉴 2~3개 요약 추천(유통기한 짧은 재료 우선) → 선택 시 상세 레시피.
- 사용자: "있는 재료로 아이 반찬 하나 만들어줘"
  → `아이선호도` 높은 재료 중심, 순한 간, 알레르기 표시한 상세 레시피 1개 생성.
- 사용자: "우유 알레르기 있는데 아침 메뉴 추천"
  → `알레르기`에 `우유` 포함 재료 제외 후 추천.
