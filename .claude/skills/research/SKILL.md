# 관심분야 자동 리서치 스킬

주제만 주면 **① 요구사항 확인 → ② 브라우저로 사이트 3곳 탐색 → ③ 고정 포맷 md 정리 → ④ 노션 공유**
까지 자동으로 수행하는 스킬입니다. (예시 결과물: `quest/week_5/5_my_agent_자동리서치_스킬/research.md`)

---

## ① 주제·키워드를 먼저 묻기 (건너뛰지 말 것)

리서치를 시작하기 전에 아래를 확인합니다. **요청에 이미 있으면 그대로 사용**하고, 빠진 것만 물어봅니다.

- **주제 / 관심분야** — 무엇을 조사할지 (예: "AI 부트캠프 훈련생 모집")
- **핵심 키워드 / 제외 조건** — 무엇을 남기고 무엇을 뺄지 (예: "채용 말고 훈련생 모집만")
- **탐색할 사이트 3곳** — 지정이 없으면 주제에 맞는 대표 사이트 3곳을 **내가 제안하고 확인**받기
- **노션 위치** — 특정 페이지 하위에 넣을지, 새 페이지로 만들지

> 여러 개를 한 번에 물어야 하면 `AskUserQuestion` 도구를 쓰면 깔끔합니다. 답이 이미 명확하면 바로 진행.

---

## ② 브라우저 MCP로 사이트 3곳 탐색

`mcp__playwright__*` (Playwright) 도구로 각 사이트를 직접 엽니다. (도구 스키마는 `ToolSearch`로 먼저 로드)

핵심 원칙:

- **스냅샷이 너무 크면** `browser_snapshot` 결과가 파일로 저장됩니다 → 통째로 읽지 말고 `Grep`으로 필요한 부분만 검색.
- **동적/가상 스크롤 리스트**(React 가상화 등)는 DOM 스크래핑이 화면에 보이는 몇 개만 잡습니다. 대신:
  - `browser_evaluate` 안에서 **같은 출처로 `fetch()`** 하면 CORS 없이 사이트 내부 API·데이터를 그대로 받습니다 (`{ credentials: 'include' }`).
  - `browser_network_requests`로 리스트를 채우는 **내부 API 엔드포인트**를 먼저 찾은 뒤 그 API를 직접 호출.
  - 상세 페이지의 `og:title` / `og:description`, 또는 API의 title 필드에서 **정확한 항목명 + 원본 링크**를 추출.
- **제외 대상**(예: 채용 공고)은 키워드 정규식으로 걸러내고, 원하는 항목만 남깁니다.
- 항목마다 **이름 + 원본 링크(모집/상세 URL) + 한 줄 설명**을 반드시 확보합니다.
- 끝나면 `browser_close`.

> 특정 채용·부트캠프 사이트(boottent / jasoseol / saramin)의 구체적 스크래핑 레시피가 필요하면
> 프로젝트 메모리 `bootcamp-recruit-site-scraping` 을 참고하세요.

산출물: 사이트(출처)별로 "항목명 + 링크 + 한 줄 설명" 목록.

## ③ md로 정리 — 고정 포맷

`research.md`(또는 주제에 맞는 파일명)로 **아래 5개 요소를 반드시 포함**해 작성합니다:
**제목 / 수집일 / 출처별 핵심 / 한 줄 요약 / 다음 액션**.

```markdown
# {제목}

- **수집일:** YYYY-MM-DD
- **주제·키워드:** {무엇을 조사, 무엇을 제외}

## 출처별 핵심

### 1. {사이트1}
- {항목명} — {링크} — {한 줄 설명}
- ...

### 2. {사이트2}
- {항목명} — {링크} — {한 줄 설명}

### 3. {사이트3}
- {항목명} — {링크} — {한 줄 설명}

## 한 줄 요약
> {전체 결과를 한 문장으로}

## 다음 액션
- [ ] {후속으로 할 일 1}
- [ ] {후속으로 할 일 2}
```

> 항목이 많으면 "출처별 핵심"을 표(`| 항목명 | 링크 | 설명 |`)로 써도 됩니다 — 노션에서도 정상 렌더링됩니다.

---

## ④ 노션에 공유

`mcp__notion__notion-create-pages`로 업로드합니다.

- **제목은 `properties.title`**, 본문은 `content`(Notion 마크다운). 본문 맨 위에 제목(H1)을 다시 넣지 말 것.
- **표는 GFM 파이프 테이블 그대로** 넣으면 노션이 자동으로 표로 변환합니다.
- 부모 페이지를 지정받았으면 `parent.page_id`로, 없으면 `parent` 생략(개인 워크스페이스 페이지로 생성).
- 업로드 후 **`notion-fetch`로 렌더링을 확인**하고, 반환된 페이지 URL을 사용자에게 전달합니다.
